Laboratorio: Mi Primer Sitio Web Estático con S3

Al finalizar este ejercicio, habrás desplegado una página web estática simple, accesible globalmente a través de Amazon S3.

Conceptos a Practicar:

  • Creación y configuración de Buckets S3.
  • Gestión de Acceso Público y Políticas de Bucket.
  • Habilitación del Alojamiento de Sitios Web Estáticos.
  • Versionado de Objetos como mecanismo de protección.

Parte 1: La Fundación - Crear el Bucket

El primer paso en S3 es siempre crear un "bucket". Piensa en un bucket como una carpeta raíz en la nube donde vivirán todos los archivos de tu proyecto.

  1. Navegar a la Consola de S3: Inicia sesión en tu Consola de AWS, busca S3 y haz clic en "Crear bucket".
  2. Configurar los Detalles del Bucket:
    • Nombre del bucket: Debe ser único a nivel mundial. Usa un formato como: mi-sitio-web-YYYYMMDD-iniciales.
    • Región de AWS: Selecciona la más cercana a tus usuarios, ej: us-east-1 (Norte de Virginia).
    • Configuración de propiedad de objetos: Deja la opción recomendada: ACL deshabilitadas.
    • Bloquear todo el acceso público: ¡MUY IMPORTANTE! Deja esta opción marcada por ahora. La seguridad es primero.
  3. Crear el Bucket: Revisa la configuración y haz clic en "Crear bucket".

Parte 2: Añadiendo Contenido

Un bucket vacío no es útil. Ahora cargaremos los archivos HTML que conformarán nuestro sitio web.

2.1 Preparar Archivos Locales

Crea dos archivos en tu computadora:

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>¡Nuestro Proyecto Despega!</title>
    <style>
        body { font-family: Arial, sans-serif; text-align: center; margin-top: 100px; background-color: #f0f8ff; }
        h1 { color: #0056b3; }
    </style>
</head>
<body>
    <h1>¡Próximamente!</h1>
    <p>Nuestro increíble proyecto está en construcción.</p>
    <p>Versión 1.0</p>
</body>
</html>
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Página no Encontrada</title>
</head>
<body>
    <h1>Oops... Algo salió mal.</h1>
    <p>No pudimos encontrar la página que buscas.</p>
</body>
</html>

2.2 Cargar y Verificar

  1. En tu bucket, haz clic en "Cargar" y añade los dos archivos.
  2. Una vez cargados, haz clic en index.html y copia la "URL del objeto".
  3. Pégala en tu navegador. Deberías ver un error "Access Denied". ¡Perfecto! La seguridad funciona.

Parte 3: Seguridad y Acceso Público

Para que el sitio sea visible, debemos desactivar el bloqueo general y crear una política de bucket que solo permita la lectura pública.

  1. Desactivar Bloqueo de Acceso Público:
    • Ve a la pestaña "Permisos" de tu bucket.
    • En "Bloquear acceso público", haz clic en "Editar".
    • Desmarca "Bloquear *todo* el acceso público", escribe confirmar y guarda los cambios.
  2. Crear una Política de Bucket:

    En la misma pestaña de "Permisos", edita la "Política de bucket". Pega el siguiente JSON, reemplazando NOMBRE_DE_TU_BUCKET con el nombre real de tu bucket.

    Análisis de la Política

    • "Effect": "Allow": Estamos creando una regla de permiso.
    • "Principal": "*": El asterisco significa "cualquiera" o anónimo.
    • "Action": "s3:GetObject": La acción permitida es solo leer/descargar.
    • "Resource": ".../*": La regla se aplica a todos los objetos (`/*`) dentro de tu bucket.
    {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::NOMBRE_DE_TU_BUCKET/*"
        }
    ]
}
  3. Probar de Nuevo: Vuelve a abrir la URL del objeto index.html. ¡Ahora deberías ver tu página web!

Parte 4: Habilitar el Alojamiento de Sitios Web

S3 aún no se comporta como un servidor web (no conoce el archivo de índice o error). Vamos a configurarlo.

  1. Ve a la pestaña "Propiedades" de tu bucket y busca "Alojamiento de sitios web estáticos". Haz clic en "Editar".
  2. Selecciona "Habilitar".
    • Documento de índice: index.html
    • Documento de error: error.html
  3. Guarda los cambios. Ahora, en la misma sección, encontrarás el endpoint del sitio web del bucket. Esta es la URL oficial de tu sitio. ¡Pruébala!

Parte 5: Versionado como Red de Seguridad

El versionado te protege contra eliminaciones o sobrescrituras accidentales.

  1. En la pestaña "Propiedades", edita "Versionado del bucket" y selecciona "Habilitar".
  2. Simula un cambio: Modifica tu index.html local (ej: cambia "Versión 1.0" por "Versión 2.0") y súbelo de nuevo, sobrescribiendo el anterior.
  3. Simula un borrado: Ahora, selecciona index.html en la consola y elimínalo. Tu sitio web debería mostrar un error.
  4. Recupera tus datos: En la pestaña "Objetos", activa el interruptor "Mostrar versiones". Verás todo el historial. Para restaurar, simplemente selecciona el "marcador de eliminación" y elimínalo. La versión más reciente (v2.0) volverá a ser la actual.

Parte 6: Limpieza de Recursos

Es fundamental en la nube eliminar siempre los recursos que no estás utilizando para evitar costos.

  1. Vaciar el Bucket: S3 no te permitirá eliminar un bucket que no esté vacío. En la página de tu bucket, haz clic en "Vaciar". Confirma la acción para eliminar permanentemente todos los objetos y sus versiones.
  2. Eliminar el Bucket: Una vez vacío, vuelve a la lista principal de buckets, selecciona el tuyo y haz clic en "Eliminar". Confirma escribiendo el nombre del bucket.
